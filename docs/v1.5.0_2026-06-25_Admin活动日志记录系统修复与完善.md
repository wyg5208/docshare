## 任务目标

Admin Dashboard 的 "Recently Activity" 区域始终显示空白，无法查看任何用户活动信息（登录、阅读、下载、退出、编辑等）。需要排查根因并修复，同时确保 Analytics 分析页的统计和日志功能一并完善。

## 问题诊断与分析

### 问题 1: Recent Activity 显示空白
**现象**: 访问 `/admin` 页面，"Recent Activity" 卡片始终显示 "No recent activity"。
**原因**: 系统设计了 `access_logs` 表用于记录用户活动（支持 login/logout/document_view/document_download 四种 action），但实际只有 login 和 document_download 两个事件有记录代码，document_view 和 logout 完全缺失。

### 问题 2: Login 事件未记录 user_id
**现象**: 即使有登录记录，Admin Dashboard 通过 `profiles(username, display_name)` 关联查询时返回 null，无法显示用户名。
**原因**: `login/page.tsx` 中插入 access_logs 记录时未传递 `user_id` 字段，导致该列为 NULL，无法与 profiles 表关联。

### 问题 3: document_view 事件完全缺失
**现象**: 用户浏览文档时从未写入 access_logs，导致 Total Views 统计始终为 0。
**原因**: 文档详情页 `doc/[slug]/page.tsx` 未实现 document_view 事件记录逻辑。

### 问题 4: logout 事件完全缺失
**现象**: 用户退出登录时从未写入 access_logs。
**原因**: `auth-provider.tsx` 的 `signOut` 函数未实现 logout 事件记录。

### 问题 5: Admin Dashboard 图标和颜色不完整
**现象**: Recent Activity 区域仅区分 login 和其他两种类型，logout 和 document_download 没有独立的图标和颜色。
**原因**: 条件渲染逻辑只覆盖了 login 和 document_view，其余统一显示 Eye 图标。

### 问题 6: Analytics 筛选器缺少 Logout 选项
**现象**: Analytics 页面的 Action 筛选下拉框只有 Login / Document Views / Downloads 三个选项。
**原因**: 筛选器未包含 Logout 选项，无法单独过滤退出事件。

## 解决方案与实施

### 1. 修复 Login 事件 user_id 缺失（login/page.tsx）

在登录成功后的 access_logs 插入操作中补充 `user_id` 字段：

```typescript
// 修复前：未传 user_id
await supabase.from("access_logs").insert({
  action: "login",
  metadata: { method: "email" },
});

// 修复后：显式传递 user_id
await supabase.from("access_logs").insert({
  user_id: user!.id,
  action: "login",
  metadata: { method: "email" },
});
```

### 2. 新增 document_view 事件记录（doc/[slug]/page.tsx）

在文档详情页的服务端组件中，用户访问文档时记录浏览事件并自增 view_count：

```typescript
// 获取当前用户并记录 document_view
const { data: { user: currentUser } } = await supabase.auth.getUser();
if (currentUser) {
  await Promise.all([
    supabase.from("access_logs").insert({
      user_id: currentUser.id,
      action: "document_view",
      document_id: document.id,
    }),
    supabase
      .from("documents")
      .update({ view_count: (document.view_count || 0) + 1 })
      .eq("id", document.id),
  ]);
}
```

同时将原有的 `getUser()` 调用与权限检查合并，避免重复请求。

### 3. 新增 logout 事件记录（auth-provider.tsx）

在用户退出登录前记录 logout 事件（必须在 signOut 之前执行，因为退出后 session 失效无法写入）：

```typescript
const signOut = async () => {
  // 在 signOut 之前记录 logout 事件
  if (user) {
    await supabase.from("access_logs").insert({
      user_id: user.id,
      action: "logout",
    });
  }
  await supabase.auth.signOut();
  setUser(null);
  setProfile(null);
};
```

### 4. 完善 Admin Dashboard 活动图标和颜色（admin/page.tsx）

为 4 种 action 类型分别配置独立的图标和颜色：

| Action | 图标 | 颜色 |
|--------|------|------|
| `login` | LogIn | 绿色 (bg-green-100 text-green-600) |
| `logout` | LogOut | 灰色 (bg-gray-100 text-gray-600) |
| `document_view` | Eye | 蓝色 (bg-blue-100 text-blue-600) |
| `document_download` | Download | 橙色 (bg-orange-100 text-orange-600) |

新增 `LogIn`、`LogOut` 图标导入，替换原来不恰当的 `Users` 图标。

### 5. Analytics 筛选器增加 Logout 选项（admin/analytics/page.tsx）

在 Action 筛选下拉框中新增 `<option value="logout">Logout</option>` 选项。

## 技术实现细节

### 事件记录架构

系统采用客户端 Supabase 直接写入 `access_logs` 表的方式，利用 RLS 策略控制访问：

- **INSERT 策略**: `WITH CHECK (auth.role() = 'authenticated')` — 所有已认证用户均可写入
- **SELECT 策略**: Admin 通过 `is_admin()` 查看所有日志，普通用户仅查看自己的日志（`user_id = auth.uid()`）
- **数据库表**: `access_logs` 包含 user_id、action（枚举类型）、document_id、ip_address、user_agent、metadata、created_at 字段
- **索引优化**: 按 user_id、action、created_at DESC 建立索引

### 事件记录触发点

| 触发点 | 文件 | 事件类型 |
|--------|------|---------|
| 用户成功登录 | `src/app/(auth)/login/page.tsx` | `login` |
| 用户退出登录 | `src/components/auth-provider.tsx` | `logout` |
| 用户浏览文档 | `src/app/doc/[slug]/page.tsx` | `document_view` |
| 用户下载文档 | `src/components/document/document-actions.tsx` | `document_download` |

## 经验教训

### 1. 功能设计需端到端验证
`access_logs` 表和 RLS 策略在初始设计中已完整定义，但事件记录代码只实现了一部分（login + download），document_view 和 logout 遗漏。功能设计完成后应逐一检查所有 action 类型是否都有对应的记录代码。

### 2. 外键关联需显式传递关键 ID
`access_logs` 表的 `user_id` 字段没有 DEFAULT 值，插入时必须显式传递。遗漏 user_id 会导致关联查询返回 null，影响数据展示。

### 3. 退出事件需在 signOut 之前记录
logout 事件必须在 `supabase.auth.signOut()` 之前写入，因为退出后用户 session 失效，无法再进行数据库写入操作。
