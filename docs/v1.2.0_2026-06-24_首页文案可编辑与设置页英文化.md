## 任务目标

1. 让 Admin 用户可以在后台自定义首页 Hero 文案（标题、强调色标题、副标题），无需修改代码。
2. 修复 `/settings` 页面 Profile Picture 区域"选择文件 / 未选择任何文件"按钮与状态显示中文的问题，强制英文 UI。

## 问题诊断与分析

### 问题 1: 首页 Hero 文案硬编码

**现象**：
- 首页大标题 `Your Documents, Organized & Accessible` 与副标题 `A modern platform to publish, organize, and share your documents...` 直接写在 `src/app/page.tsx` 里。
- 任何文案调整都需要开发者改代码 + 重新部署，非技术 Admin 无法自助修改。

**根因**：
- 缺少站点级可编辑文案存储与读取机制。

### 问题 2: Settings 页文件上传控件中文

**现象**：
- `/settings` 页面 Profile Picture 区域，"选择文件"按钮和"未选择任何文件"提示文字为中文。
- 与项目其它所有 UI 均为英文的风格不一致。

**根因**：
- 原生 `<input type="file">` 的按钮文案由浏览器/系统语言决定，无法通过 CSS 或 HTML 属性强制改为英文。
- 项目代码本身 `src/` 目录扫描 `[\u4e00-\u9fff]` 为 0 处匹配，无中文 UI 文案遗漏。

## 解决方案与实施

### 方案 A：首页文案可编辑化

#### 1. 数据库层：`site_settings` 键值表

新建迁移 `supabase/migrations/004_site_settings.sql`，设计为通用 key-value 表，便于后续扩展 footer / about 等更多可编辑字段：

```sql
CREATE TABLE IF NOT EXISTS public.site_settings (
  key         TEXT PRIMARY KEY,
  value       TEXT,
  updated_by  UUID REFERENCES public.profiles(id) ON DELETE SET NULL,
  updated_at  TIMESTAMPTZ DEFAULT now() NOT NULL
);

ALTER TABLE public.site_settings ENABLE ROW LEVEL SECURITY;

-- 任意访问者（含未登录）可读，用于首页 SSR
CREATE POLICY "Public read site settings"
  ON public.site_settings FOR SELECT USING (true);

-- 仅 admin 可写
CREATE POLICY "Admins manage site settings"
  ON public.site_settings FOR ALL
  USING (is_admin()) WITH CHECK (is_admin());
```

种子数据：

```sql
INSERT INTO public.site_settings (key, value) VALUES
  ('hero_title_main',      'Your Documents,'),
  ('hero_title_highlight', 'Organized & Accessible'),
  ('hero_subtitle',        'A modern platform to publish, organize, and share your documents. Support for PDFs, images, videos, audio files, and more.')
ON CONFLICT (key) DO NOTHING;
```

**设计要点**：
- 键值表设计便于扩展更多可编辑字段
- 公开 SELECT（含未登录用户访问首页）
- 仅 admin 可写（依赖 `is_admin()` 函数）
- 幂等迁移：`IF NOT EXISTS` + `DROP POLICY IF EXISTS` + `ON CONFLICT DO NOTHING`

#### 2. 首页读取 + 降级默认值

`src/app/page.tsx` 在 Server Component 中读取 `site_settings`，未配置时降级为原硬编码默认值：

```tsx
const { data: settings } = await supabase
  .from("site_settings")
  .select("key, value")
  .in("key", ["hero_title_main", "hero_title_highlight", "hero_subtitle"]);

const settingsMap = Object.fromEntries(
  (settings || []).map((s) => [s.key, s.value])
);

const heroTitleMain = settingsMap.hero_title_main || "Your Documents,";
const heroTitleHighlight = settingsMap.hero_title_highlight || "Organized & Accessible";
const heroSubtitle = settingsMap.hero_subtitle || "A modern platform to publish, organize, and share your documents. Support for PDFs, images, videos, audio files, and more.";
```

渲染：

```tsx
<h1>
  {heroTitleMain}{" "}
  <span className="text-primary">{heroTitleHighlight}</span>
</h1>
<p className="... whitespace-pre-line">{heroSubtitle}</p>
```

**降级保护**：DB 读不到时使用硬编码默认值，避免迁移未执行导致首页空白。

### 方案 B：Settings 页文件上传控件英文化

`src/app/(dashboard)/settings/page.tsx` 改造：

```tsx
<input ref={fileInputRef} type="file" accept="image/*" onChange={handleAvatarUpload} className="hidden" />
<Button type="button" variant="outline" onClick={() => fileInputRef.current?.click()} disabled={uploadingAvatar}>
  {uploadingAvatar && <Loader2 className="mr-2 h-4 w-4 animate-spin" />}
  Choose File
</Button>
<span className="text-sm text-muted-foreground truncate max-w-[200px]">
  {selectedFileName || "No file chosen"}

```

**全局中文检查**：

## 经验教训

### 1. 原生 HTML 控件的本地化陷阱
浏览器原生 `<input type="file">` 的按钮文案由浏览器/系统语言决定，无法通过 CSS 或 HTML 属性强制改为英文。解决方式是隐藏原生控件，用自定义按钮 + `useRef` 触发。

### 2. 可编辑文案存储设计
使用通用 key-value 表而非为每个字段建列，便于后续扩展 footer / about / FAQ 等更多可编辑字段，无需再改表结构。

### 3. 降级默认值的必要性
DB 读不到时使用硬编码默认值，避免迁移未执行导致首页空白。这种"防御性降级"是站点配置类功能的通用最佳实践。

### 4. RLS 公开读 + admin 写
对于站点级公开配置，使用 `USING (true)` 允许任何访问者（含未登录）读取，配合 `is_admin()` 限制写入，既保证首页 SSR 可用，又保证安全性。
