## 任务目标

为 DocShare 网站新增两个炫酷主题：**黑夜模式（Dark Night）**和**墨绿科幻主题（Sci-Fi Green）**，使网站从单一白色主题扩展为三主题可切换。用户可通过 Header 右侧的主题切换器一键切换，主题选择自动持久化到 localStorage。

## 问题诊断与分析

### 现状分析
**现象**: 网站仅有白色主题，无暗色或彩色主题可选
**原因**: 虽然项目已集成 `next-themes` v0.4.6 和 Tailwind CSS v4 的 CSS 变量体系，但仅定义了 Light 模式的变量值，未扩展 Dark/Matrix 主题变量

### 有利条件
- `next-themes` 已安装并配置 `attribute="class"`
- `@custom-variant dark (&:is(.dark *))` 已在 globals.css 中预定义
- 所有 UI 组件（Button、Card、Input 等）均使用语义化 CSS 变量类名，零硬编码颜色
- `suppressHydrationWarning` 已在 html 标签设置

## 解决方案与实施

### 1. CSS 变量扩展（globals.css）

在 `@layer base` 中新增两套完整的 CSS 变量覆盖：

**Dark 主题（`.dark` 选择器）**：
- 深蓝黑色底（`hsl(222.2 84% 4.9%)`）
- 蓝色主色调（`hsl(217.2 91.2% 59.8%)`）
- 高对比度白色文字
- 覆盖全部 38 个 CSS 变量（含 sidebar、chart 等）

**Matrix 科幻主题（`.matrix` 选择器）**：
- 深墨绿底（`hsl(160 30% 5%)`）
- 翠绿霓虹主色（`hsl(142 71% 45%)`）
- 科幻 Matrix 电影风格
- 覆盖全部 38 个 CSS 变量

同时新增 `@custom-variant matrix (&:is(.matrix *))` 自定义变体选择器，使 Tailwind 的 `matrix:` 前缀类可用。

### 2. ThemeProvider 配置更新（layout.tsx）

在根布局的 ThemeProvider 中添加 `themes` 属性，显式声明支持的主题列表：

```tsx
<ThemeProvider
  attribute="class"
  defaultTheme="light"
  themes={["light", "dark", "matrix"]}
  enableSystem={false}
  disableTransitionOnChange
>
```

### 3. 主题切换器组件（theme-switcher.tsx）

新建 `src/components/theme-switcher.tsx` 组件：
- 下拉菜单式选择器，3 个主题选项各带图标+名称+预览色块
- 图标：Sun（Light）、Moon（Dark Night）、Leaf（Sci-Fi Green）
- 使用 `useTheme()` hook 获取/设置当前主题
- `mounted` 守卫防止 SSR hydration 闪烁
- 点击外部自动关闭下拉菜单

### 4. Header 集成（header.tsx）

在 Header 右侧区域（User Menu 前）插入 `<ThemeSwitcher />` 组件，使主题切换在所有页面全局可用。

### 5. 硬编码颜色适配

以下区域存在硬编码 Tailwind 颜色类，逐一添加 `dark:` 和 `matrix:` 前缀适配：

| 文件 | 位置 | 问题 | 修复 |
|------|------|------|------|
| `analytics/page.tsx` | 统计卡片图标背景 | `bg-blue-50` 等浅色背景 | 添加 `dark:bg-blue-900/30 matrix:bg-blue-900/30` |
| `analytics/page.tsx` | 活动日志颜色映射 | `bg-green-100 text-green-600` | 添加 dark/matrix 深色变体 |
| `users/page.tsx` | 有效期状态徽章 | `text-red-600 bg-red-50` | 添加 dark/matrix 深色变体 |
| `toast.tsx` | 边框颜色 | `border-red-200` | 添加 dark/matrix 深色边框 |
| `login/page.tsx` | 警告框 | 已有 dark: 前缀 | 补充 matrix: 前缀 |

**保持不变**的区域：Toast 图标色（text-green-500 等）和文件类型图标色（file-icon.tsx），这些是功能性语义色，在深色背景下可见度良好。

### 6. 过渡动画（globals.css）

为 `body` 添加 `transition: background-color 0.3s ease, color 0.3s ease`，实现主题切换时的平滑过渡效果。

## 技术实现细节

### 核心架构
```
ThemeProvider (next-themes)
  ↓ attribute="class"
<html class="light|dark|matrix">
  ↓ CSS 变量覆盖
@layer base { .dark { --color-*: ... } .matrix { --color-*: ... } }
  ↓ 所有组件自动响应
bg-background, text-foreground, bg-primary, ...
```

### CSS 变量覆盖模式
```css
@layer base {
  .dark {
    --color-background: hsl(222.2 84% 4.9%);
    --color-foreground: hsl(210 40% 98%);
    /* ... 38 个变量全覆盖 ... */
  }
  .matrix {
    --color-background: hsl(160 30% 5%);
    --color-foreground: hsl(142 70% 85%);
    /* ... 38 个变量全覆盖 ... */
  }
}
```

## 经验教训

### 1. CSS 变量驱动的全局主题方案
项目已采用语义化 CSS 变量体系（bg-background、text-foreground 等），使得主题切换只需在根层覆盖变量值，无需逐组件修改，极大降低了多主题的实现成本。

### 2. Tailwind v4 的 @custom-variant 能力
Tailwind v4 支持自定义变体选择器，使得非标准主题名（如 `matrix`）也能使用 `matrix:` 前缀语法，与内置的 `dark:` 前缀同等灵活。

### 3. 硬编码颜色是主题化的主要障碍
虽然核心 UI 组件零硬编码颜色，但 Admin 页面（Analytics、Users）和 Toast 组件中存在少量直接 Tailwind 颜色类，需要逐一排查并添加 dark/matrix 前缀适配。
