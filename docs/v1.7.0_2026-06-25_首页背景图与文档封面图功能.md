- **日期**: 2026-06-25

## 任务目标

为 DocShare 平台增加视觉增强功能：
1. **首页 Hero 背景图**：当前首页 Hero 区域仅使用 CSS 渐变背景，缺乏视觉冲击力。新增管理员可配置的背景图功能，支持三种模式：CSS 渐变（默认）、Unsplash 预设图片、自定义上传。
2. **文档卡片封面图**：当前 `/browse` 页面的文档卡片仅显示图标 + 文字信息。新增三种封面模式：默认图标、纯色背景 + 图标、预设/自定义图片，提升文档浏览页的视觉丰富度。

## 问题诊断与分析

### 问题 1: 首页缺乏视觉吸引力

**现象**：
- 首页 Hero 区域仅有 `bg-gradient-to-b from-primary/5 via-background to-background` 的淡色渐变
- 作为文档共享平台，首页缺乏与教育培训场景相关的视觉元素

**原因**：
- 初始设计以功能完整性优先，未考虑视觉背景图需求
- `site_settings` 表已有可扩展的 key-value 架构，但尚未包含背景图相关设置

### 问题 2: 文档卡片信息密度低

**现象**：
- 文档卡片顶部区域仅显示 `FileIcon`（文件类型图标）
- 非图片类文档全部显示相同的 `bg-muted/50` 灰色背景
- 在 `/browse` 页面浏览时，所有卡片外观高度一致，缺乏区分度

**原因**：
- `documents` 表虽有 `thumbnail_url` 字段，但仅用于图片类文档的缩略图
- 缺少 `cover_color` 字段用于纯色封面模式

## 解决方案与实施

### 1. 数据库 Schema 扩展

**新增迁移文件**：`supabase/migrations/008_cover_images.sql`

- `documents` 表新增 `cover_color TEXT` 列（可为空，默认 NULL）
- `site_settings` 表插入两个新键值：
  - `hero_bg_type` = `'gradient'`（可选值：`gradient` / `preset` / `custom`）
  - `hero_bg_image` = `''`（存储预设图 ID 或自定义图片 URL）

### 2. 预设图片常量配置

**新增文件**：`src/lib/preset-images.ts`

定义了三个数据集：
- `HERO_PRESET_IMAGES`：12 张教育/校园主题的 Unsplash 宽幅图片（毕业、校园、图书馆、教室等）
- `DOCUMENT_COVER_PRESETS`：12 张通用文档主题的 Unsplash 图片（书籍、笔记本、办公等）
- `COVER_COLORS`：12 种预设颜色（蓝、靛蓝、紫、粉、红、橙、琥珀、绿、青、蓝绿、石板灰、锌灰）

每张图片包含 `id`、`url`（全尺寸）、`thumbnail`（缩略图）、`label`（描述标签）。

### 3. 共享图片选择/上传组件

**新增文件**：`src/components/admin/image-upload-picker.tsx`

可复用的 `ImageUploadPicker` 组件，支持四种模式：
- **None**：不使用背景/封面
- **Color**：纯色背景（通过色板选择器）
- **Preset**：从预设图片网格中选择
- **Custom**：上传自定义图片到 Supabase Storage

组件特性：
- 模式切换 Tab 栏
- 预设图片网格带选中高亮和标签
- 自定义上传支持 5MB 限制和图片格式校验
- 实时预览当前选中效果

### 4. 首页 Hero Section 背景图支持

**修改文件**：`src/app/page.tsx`

- 从 `site_settings` 额外获取 `hero_bg_type` 和 `hero_bg_image`
- 根据 `hero_bg_type` 渲染不同背景层：
  - `gradient`：保持原有 CSS 渐变
  - `preset`：查找 `HERO_PRESET_IMAGES` 中对应 ID 的 URL
  - `custom`：直接使用 `hero_bg_image` 中的 Supabase Storage URL
- 背景图上叠加 `bg-black/50` 半透明遮罩确保文字可读性
- 有背景图时文字自动切换为白色，按钮使用 `secondary` 变体

### 5. 文档卡片封面图支持

**修改文件**：`src/components/document/document-card.tsx`

- 新增 `hasCoverImage` 和 `hasCoverColor` 判断逻辑
- 渲染优先级：`thumbnail_url` > `cover_color` > 默认样式
- 纯色模式下 FileIcon 自动变为白色
- 图片模式下添加 `loading="lazy"` 懒加载

### 6. Admin Hero 背景图设置 UI

**修改文件**：`src/app/(admin)/admin/site-settings/page.tsx`

- DEFAULTS 新增 `hero_bg_type: "gradient"` 和 `hero_bg_image: ""`
- Hero Tab 中新增 "Hero Background" Card，集成 `ImageUploadPicker` 组件
- Preview 卡片实时展示背景效果（含遮罩和文字颜色变化）
- Save/Reset 按钮保存 `hero_bg_type` 和 `hero_bg_image`

### 7. Admin 文档封面图编辑 UI

**修改文件**：
- `src/app/(admin)/admin/documents/new/page.tsx`
- `src/app/(admin)/admin/documents/[id]/edit/page.tsx`

- 新增 "Cover Image" Card（位于 File 和 Details 之间）
- 集成 `ImageUploadPicker`，支持 None/Color/Preset/Custom 四种模式
- 提交时根据 `coverMode` 写入对应字段：
  - `color` 模式 → 写入 `cover_color`，清空 `thumbnail_url`
  - `preset`/`custom` 模式 → 写入 `thumbnail_url`，清空 `cover_color`
  - `none` 模式 → 清空两者
- 编辑页自动恢复已保存的封面状态

### 8. 图片域名白名单

**修改文件**：`next.config.ts`

- 添加 `images.unsplash.com` 到 `remotePatterns`，允许 Next.js 优化 Unsplash 预设图片

## 技术实现细节

### Hero 背景图渲染逻辑

```tsx
// src/app/page.tsx
const heroBgType = settingsMap.hero_bg_type || "gradient";
const heroBgImageRaw = settingsMap.hero_bg_image || "";
const heroImage = heroBgType === "preset"
  ? HERO_PRESET_IMAGES.find((p) => p.id === heroBgImageRaw)?.url || heroBgImageRaw
  : heroBgImageRaw;
const hasHeroBg = heroBgType !== "gradient" && !!heroImage;

// 渲染
<section className="relative overflow-hidden">
  {hasHeroBg ? (
    <div className="absolute inset-0">
      <img src={heroImage} className="w-full h-full object-cover" loading="eager" />
      <div className="absolute inset-0 bg-black/50" />
    </div>
  ) : (
    <div className="absolute inset-0 bg-gradient-to-b from-primary/5 via-background to-background" />
  )}
  {/* 文字内容根据 hasHeroBg 切换颜色 */}
</section>
```

### 文档卡片封面优先级

```tsx
// src/components/document/document-card.tsx
const hasCoverImage = !!document.thumbnail_url;
const hasCoverColor = !!document.cover_color && !hasCoverImage;

<div
  style={hasCoverColor ? { backgroundColor: document.cover_color! } : undefined}
  className={!hasCoverImage && !hasCoverColor ? "bg-muted/50" : ""}
>
  {hasCoverImage ? (
    <img src={document.thumbnail_url!} loading="lazy" />
  ) : (
    <FileIcon className={hasCoverColor ? "text-white/90" : "text-muted-foreground"} />
  )}
</div>
```

## 经验教训

### 1. 复用现有字段优于新增
`documents` 表已有 `thumbnail_url` 字段，直接复用作为文档封面图片 URL，避免引入新的图片字段。仅新增 `cover_color` 一个列即可支持纯色模式。

### 2. site_settings key-value 架构的扩展性
利用已有的 `site_settings` 表存储 Hero 背景设置，新增两个 key 即可，无需修改表结构或 RLS 策略。

### 3. 预设图片使用 CDN 而非本地存储
将数十张高清图片放入 `public/` 目录会使 git 仓库膨胀 50MB+。使用 Unsplash CDN URL 零存储成本，且图片质量由 Unsplash 保证。

### 4. 共享组件减少重复代码
`ImageUploadPicker` 组件同时服务于 Hero 背景和文档封面两个场景，通过 Props 灵活配置预设列表、颜色板、上传路径等参数。
