## 任务目标

修复用户反馈的"登录后进入某些页面导航栏消失"问题，确保站点导航栏（Header）在所有用户可达页面中保持一致出现，提升导航连贯性与用户体验。

## 问题诊断与分析

### 问题 1: 部分公共页面缺失导航栏

**现象**：
- 用户登录后，从首页点击进入 `/browse`（浏览）、`/search`（搜索）、`/tags`（标签）等页面时，顶部导航栏（Header）消失。
- 在分类详情页 `/browse/[slug]` 和标签详情页 `/tags/[slug]` 也存在同样问题。
- 用户主观感受是"登录后导航栏丢失"，但实际上该问题与登录态本身无关。

**根本原因**：

DocShare 项目使用 Next.js 15 App Router，采用**路由组（Route Group）**组织布局：

```
src/app/
├── (admin)/      # admin layout，含 AdminHeader
├── (auth)/       # auth layout
├── (dashboard)/  # dashboard layout，含 Header
├── (public)/     # public layout，含 Header + Footer
├── browse/       # ⚠️ 未在任何路由组内，仅继承根 layout
├── search/       # ⚠️ 未在任何路由组内，仅继承根 layout
├── tags/         # ⚠️ 未在任何路由组内，仅继承根 layout
├── doc/          # 自身页面内已包裹 Header（无问题）
└── layout.tsx    # 根 layout —— 仅提供 <html><body>，**不含 Header**
```

经核查，根 layout (`src/app/layout.tsx`) 只负责提供 `<html>`、`<body>`、`ThemeProvider`、`AuthProvider` 等全局上下文，**并不渲染 Header/Footer**。

因此，凡是不在 `(public)` 等路由组内的页面，都会缺失导航栏。受影响的 5 个页面恰好是用户浏览站点时常进入的"逛站点"主路径，导致用户误以为是"登录后才消失"。

| 页面 | 路由组 | 原本是否有 Header | 状态 |
|---|---|---|---|
| `/`（首页） | `(public)` 内同名页 | ✅ 有 | 正常 |
| `/browse` | ❌ 无 | ❌ 无 | 🔴 缺失 |
| `/browse/[slug]` | ❌ 无 | ❌ 无 | 🔴 缺失 |
| `/search` | ❌ 无 | ❌ 无 | 🔴 缺失 |
| `/tags` | ❌ 无 | ❌ 无 | 🔴 缺失 |
| `/tags/[slug]` | ❌ 无 | ❌ 无 | 🔴 缺失 |
| `/doc/[id]` | ❌ 无 | ✅ 页面内已包裹 | 正常 |
| `/admin/*` | `(admin)` | ✅ AdminHeader | 正常 |
| `/bookmarks` / `/settings` | `(dashboard)` | ✅ 有 | 正常 |

## 解决方案与实施

### 方案选型

**候选方案 A**：将 Header/Footer 上移至根 layout —— ❌ 不合适，因为 `(admin)` 路由组使用 AdminHeader，不能与公共 Header 共存。

**候选方案 B**：将 5 个页面目录迁移到 `(public)` 路由组 —— ❌ 改动大，且涉及 URL 不变但目录树重组，可能影响其他引用。

**候选方案 C**（✅ 采用）：在受影响的 5 个页面内直接包裹 `<Header />` 和 `<Footer />`。改动最小，符合现有项目中 `/doc/[id]` 等独立页面的处理惯例。

### 实施细节

对每个受影响页面进行如下修改：

1. **新增 import**：
   ```tsx
   import { Header } from "@/components/layout/header";
   import { Footer } from "@/components/layout/footer";
   ```

2. **包裹页面结构**：将原顶层 `<div>` 改为 `<><Header /><main>...</main><Footer /></>` 三段式结构，保持原有页面主体内容不变。

修改后的页面结构示例：

```tsx
return (
  <>
    <Header />
    <main className="container mx-auto px-4 py-8 sm:px-6 lg:px-8">
      {/* 原有页面内容 */}
    </main>
    <Footer />
  </>
);
```

## 经验教训

### 1. 路由组（Route Group）布局陷阱
Next.js App Router 的路由组 `(group)` 是组织布局的强力工具，但**路由组的 layout 不会自动应用到组外的同名兄弟目录**。当存在多个不同 Header 的路由组（如本项目的 `(public)` vs `(admin)`）时，不能简单地把 Header 上移至根 layout。

### 2. 目录组织建议
理想做法是把所有"公共可访问页面"集中放在 `(public)` 路由组内统一管理。本次为最小化改动选择了"页面内包裹"方案，后续如有重构机会，建议将 `browse`、`search`、`tags` 目录整体迁入 `(public)` 路由组。

### 3. 用户反馈的解读
用户描述"登录后导航栏消失"实际上与登录态完全无关，是**路径偶然性**造成的错觉——登录后用户更可能去浏览/搜索文档，从而进入这些恰好缺失 Header 的页面。诊断时不要被表面现象带偏，要回到路由 + 布局的源头查找根因。

### 4. 注意事项
新增公共页面时，应优先考虑放入 `(public)` 路由组，或在 PR 检查清单中加入"是否包含 Header/Footer"项。
