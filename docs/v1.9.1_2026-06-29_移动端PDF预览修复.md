## 任务目标

修复移动端（iOS Safari / Android Chrome）浏览文档时无法在线阅读 PDF 的问题。将原有的 `<iframe>` 方案替换为基于 `react-pdf` 的 Canvas 渲染方案，实现真正的跨平台 PDF 阅读器，支持逐页翻页、响应式宽度、CJK 字体渲染，并在 PDF 加载失败时提供下载降级方案。

### 核心目标
1. 替换 iframe 为 react-pdf Canvas 渲染，解决移动端 PDF 无法显示的根本问题
2. 配置 PDF.js Worker（CDN），启用 CMap 支持 CJK 字体
3. 修复容器 `overflow-hidden` 导致的内容裁剪
4. 实现响应式宽度适配（ResizeObserver）
5. 添加逐页翻页导航和加载/错误状态反馈

## 问题诊断与分析

### 问题 1: 移动端 PDF 无法在线阅读

**现象**：
- 用户在 iOS Safari 或 Android Chrome 上打开文档详情页时，PDF 预览区域显示空白或仅显示首页静态图片
- 部分 Android 设备会弹出下载提示而非内嵌渲染
- 桌面端（Chrome / Edge / Firefox）显示正常

**根本原因**：

`pdf-viewer.tsx` 使用 `<iframe src={signedUrl}>` 方案渲染 PDF。这是桌面浏览器特有的能力——桌面浏览器内置了 PDF 插件（如 Chrome 的 PDF Viewer），可以在 iframe 中渲染 PDF。但移动端浏览器（iOS Safari / Android Chrome）的 WebKit/Blink 引擎**不支持在 iframe 中内嵌 PDF 渲染器**，这是平台限制而非代码 bug。

| 平台 | iframe 渲染 PDF | 结果 |
|---|---|---|
| 桌面 Chrome/Edge | ✅ 支持 | 正常显示 |
| iOS Safari | ❌ 不支持 | 空白区域或仅显示首页 |
| Android Chrome | ⚠️ 不稳定 | 可能触发下载而非内嵌渲染 |

### 问题 2: 容器 overflow-hidden 裁剪

**现象**：
- 即使 PDF 在移动端部分渲染了首页，也可能被容器裁剪导致内容不完整显示

**根本原因**：

文档详情页 `doc/[slug]/page.tsx` 的预览容器使用 `overflow-hidden` 配合 `rounded-xl` 实现圆角裁剪。在 iOS Safari 中，iframe 内容可能尝试扩展超出声明高度，被 `overflow-hidden` 截断。

### 问题 3: 视口高度单位问题

**现象**：
- 原 iframe 使用 `h-[70vh]` 固定高度

**根本原因**：

`vh` 单位在 iOS Safari 中不等于实际可视高度（地址栏显示时可视区域更小）。应使用 `dvh`（dynamic viewport height）。但替换为 react-pdf 后此问题自然消除——Canvas 渲染按页面比例自适应，无需固定高度。

### 已排除的非问题

- ✅ Supabase 签名 URL 配置正确（`Content-Disposition: inline`，无 `X-Frame-Options` 阻断）
- ✅ 无 CSP 安全头部阻止 iframe 加载
- ✅ 无混合内容问题（HTTPS 全链路）

## 解决方案与实施

### 方案选型

**候选方案 A**（✅ 采用）：使用已安装的 `react-pdf@9.2.1` 实现 Canvas 渲染
- 优点：跨平台兼容、已安装无需新增依赖、支持文本选择/链接点击、CJK 字体支持
- 缺点：需配置 Worker（CDN 加载）

**候选方案 B**：移动端检测 UserAgent，显示下载链接；桌面端保留 iframe
- 优点：改动最小
- 缺点：移动端用户体验差，需要下载后才能查看

**候选方案 C**：使用 `<object>` 或 `<embed>` 标签替代 iframe
- 优点：改动小
- 缺点：移动端兼容性同样不稳定，不如 Canvas 方案可靠

### 实施细节

#### 1. 重写 `pdf-viewer.tsx`（核心变更）

将 iframe 方案完全替换为 react-pdf Canvas 渲染：

- **Worker 配置**：使用 unpkg CDN 加载版本匹配的 Worker
  ```tsx
  pdfjs.GlobalWorkerOptions.workerSrc = `//unpkg.com/pdfjs-dist@${pdfjs.version}/build/pdf.worker.min.mjs`;
  ```

- **CJK 字体支持**：配置 CMap 和标准字体 URL
  ```tsx
  const documentOptions = {
    cMapUrl: `//unpkg.com/pdfjs-dist@${pdfjs.version}/cmaps/`,
    cMapPacked: true,
    standardFontDataUrl: `//unpkg.com/pdfjs-dist@${pdfjs.version}/standard_fonts/`,
  };
  ```

- **响应式宽度**：使用 `ResizeObserver` 监测容器宽度，`Math.min(containerWidth, 800)` 计算页面宽度
- **逐页渲染**：单页 Canvas 渲染 + 前后翻页按钮，避免一次性渲染全部页面导致性能问题
- **加载状态**：Document 和 Page 均有独立的 loading 动画
- **错误降级**：PDF 加载失败时显示错误信息和下载按钮
- **文件切换重置**：`key={file}` + `useEffect` 确保切换文档时重置页码和错误状态

#### 2. 修复容器裁剪

`doc/[slug]/page.tsx` 第 108 行：
```diff
- <div className="border rounded-xl overflow-hidden bg-muted/20 mb-6">
+ <div className="border rounded-xl overflow-auto bg-muted/20 mb-6">
```

`overflow-hidden` → `overflow-auto`：当内容未溢出时行为与 `overflow-hidden` 一致（圆角裁剪正常），当内容溢出时显示滚动条而非裁剪内容。

## 技术实现细节

### 核心代码片段

```tsx
// pdf-viewer.tsx - 响应式宽度测量
const [containerWidth, setContainerWidth] = useState<number>(0);
const containerRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  if (!containerRef.current) return;
  const updateWidth = () => {
    if (containerRef.current) {
      setContainerWidth(containerRef.current.clientWidth);
    }
  };
  updateWidth();
  const resizeObserver = new ResizeObserver(updateWidth);
  resizeObserver.observe(containerRef.current);
  return () => resizeObserver.disconnect();
}, []);

// 计算页面宽度：适配容器宽度，上限 800px
const pageWidth = containerWidth > 0 ? Math.min(containerWidth, 800) : undefined;
```

## 经验教训

### 1. 移动端 PDF 渲染的平台限制
`<iframe src="*.pdf">` 依赖浏览器内置 PDF 插件，桌面浏览器支持但移动端普遍不支持。任何需要在移动端展示 PDF 的 Web 应用，都必须使用 Canvas 方案（react-pdf / pdf.js）或提供下载备选。

### 2. 已安装但未使用的依赖
项目 `package.json` 中已声明 `react-pdf@9.2.1` 和 `pdfjs-dist@4.8.69`，但代码中从未引用——开发者在注释中写道"react-pdf requires worker setup"后放弃了。这提醒我们：已安装但未使用的依赖增加了包体积，应该及时启用或移除。

### 3. CDN Worker 加载的权衡
使用 unpkg CDN 加载 PDF.js Worker 简化了部署配置（无需 webpack/turbopack 配置），但依赖外部 CDN 可用性。如果 unpkg 不可用，PDF 预览将无法工作。未来可考虑将 Worker 文件复制到 `public/` 目录实现本地托管。

### 4. 注意事项
- `overflow-hidden` 在移动端可能裁剪 iframe 内容，改用 `overflow-auto` 更安全
- `vh` 单位在 iOS Safari 中不等于实际可视高度，应使用 `dvh` 或避免固定高度
- react-pdf 的 CSS（AnnotationLayer.css、TextLayer.css）必须在组件中导入，否则文本选择和链接点击功能异常
